The generated spec runs in an isolated `uv run pytest` subprocess with NO DB/Neo4j/live-page access in the worker — so healing MUST run INSIDE the spec where the live Playwright `page` is in scope. Add the pure verdict-override helper so a journal'd auto_heal becomes the `auto_healed` verdict (a heal is NOT a flake) — overriding classify_retry.

Purpose: HEAL-01 (live re-validation + priority-chain candidate search against the live page) and HEAL-02 (the 3 outcomes resolved live, assertions never weakened). The journal handoff is the seam plan 03 ingests.
Output: the in-spec template + the codegen wiring + the page-object accessor + the verdict-override helper + a byte-equivalence drift guard + a keyless in-spec functional proof against a live mutated page.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| in-spec subprocess → heal-journal file | The spec (untrusted in the sense it runs against a live, possibly-mutated target) writes a JSON journal under its own per-flow workspace dir. |
| live page DOM → candidate enumeration | Page content is untrusted input; candidate attrs/text are read from the live DOM. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-04 | Tampering | false heal on a non-unique live match | mitigate | The hard `page.locator(selector).count() == 1` re-validation (live, in-spec) feeds heal_outcome's uniqueness gate — 0 or >1 matches can never auto_heal. Proven by the removed-element functional case. |
| T-08-05 | Tampering | page-derived text injected into a Playwright selector | mitigate | `_to_selector` builds selectors from repo chain entries + structured live attrs; selectors use Playwright's parameterized engines (`[data-test="..."]`, `text=`, `xpath=`) — no shell, no eval of page text into code. |
| T-08-06 | Tampering | path traversal via a journal path from page text | mitigate | out_dir is passed in from the spec's KNOWN per-flow workspace path; never derived from page/DOM text (carry T-07-11). |
| T-08-07 | Elevation of Privilege | weakening an assertion to pass | mitigate | heal() and _resolve touch ONLY locator resolution; the path contains no `expect(` — a real defect raises and the test fails naturally (D-04). |
| T-08-08 | Tampering | vendored scorer drifting from the canonical engine | mitigate | The byte-equivalence drift-guard test fails the build if the in-spec copy diverges from app/services/healing. |
</threat_model>
