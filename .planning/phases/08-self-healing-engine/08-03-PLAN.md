<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| heal-journal file → worker ingest | The journal is machine-written by the (untrusted) in-spec subprocess; the worker reads + persists it. |
| ingest → page-object file rewrite | The worker mutates generated Python under the run workspace. |
| ingest → Neo4j write-back | The single-writer path mutates KG Element state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-09 | Tampering | malicious/oversized heal-journal crashing or poisoning ingest | mitigate | Tolerant bounded parse (mirrors kg/reader._loads): malformed/oversized entries skipped, never crash; required keys validated before a row is written. |
| T-08-10 | Tampering | code injection via the page-object rewrite | mitigate | Line-targeted replace keyed by attr name ONLY (the template guarantees one line per attr); ast.parse validates the result before persisting — a non-parsing rewrite raises, never written. |
| T-08-11 | Tampering | Cypher injection via journal/page text in the write-back | mitigate | append_element_history is parameterized only ($key/$history_json/$chain_json/$now); labels are kg/schema constants; routed through the single writer's read-back guard (carry T-04-14/T-05-01). |
| T-08-12 | Tampering | path traversal via a journal-supplied path | mitigate | project_root + journal path are run_id-derived via workspaces helpers; never a path from the journal body (carry T-07-11). |
| T-08-13 | DoS | binary/unbounded data in Postgres | mitigate | heal_audit stores chains as JSON + scalars only; no blobs (carry the execution-history rule). |
| T-08-14 | DoS | a down neo4j crashing the worker mid-run | mitigate | The KG write-back is best-effort (wait_for/try-except); the audit row + rewrite persist regardless. |
</threat_model>
