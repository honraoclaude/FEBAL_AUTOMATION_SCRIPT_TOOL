## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → dashboard/coverage API | role-scoped read access must be enforced server-side |
| API → Postgres + Neo4j | aggregation reads cross both stores; the graph may be down |
| QA dashboard → artifact references | run-relative paths must never become a filesystem traversal vector |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-07 | Elevation of Privilege | GET /api/dashboards/* + /api/coverage/flows | mitigate | per-route `require_role(...)` from the rbac.py matrix; the 403 matrix is asserted in test_dashboards |
| T-10-08 | Information Disclosure | a role seeing a dashboard it shouldn't | mitigate | deny-by-default — only the permitted roles pass require_role; nav-hiding (Plan 05) is not the control |
| T-10-09 | Tampering | path traversal via QA artifact references | mitigate | the service emits the RUN-RELATIVE stored `path` only; the auth-gated resolving route (Phase-7 containment guard) owns serving — Plan 05 builds the URL, never an fs path |
| T-10-10 | Denial of Service | graph-down crashing the executive dashboard | mitigate | coverage degrades to the honest 503/"graph unavailable"; the relational tiles still render — no unhandled 500 |
| T-10-11 | Information Disclosure | conflating the two coverage metrics into one misleading number | mitigate | coverage_dash is a distinct module shipping its own honest definition; ground-truth stays separate (Pitfall 5) |
