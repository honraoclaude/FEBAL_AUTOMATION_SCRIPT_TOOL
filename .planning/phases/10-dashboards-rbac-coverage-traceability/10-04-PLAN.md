<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → search API | role-scoped search; the query string is untrusted input |
| API → Elasticsearch | the search store; may be down (profile off); a derived rebuildable index |
| write paths → ES on-write index | a best-effort dual-write that must never break the Postgres source of truth |
| indexed documents | must not carry secrets/tokens (redaction before index) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-17 | Tampering | search query / ES injection | mitigate | q passed as a structured `multi_match.query` VALUE, never string-concatenated into the DSL (reader.py discipline) |
| T-10-18 | Elevation of Privilege | unauth/over-privileged search access | mitigate | router-level `require_role(...)` (all authed roles); unauth → 401; per-row drill links respect each surface's own gate |
| T-10-19 | Denial of Service | an ES outage breaking the Postgres execution/defect write | mitigate | every on-write `es.index` is swallow-and-logged (es_index_skipped) — the Postgres commit is the source of truth (Pitfall 3) |
| T-10-20 | Denial of Service | ES-down 500ing search | mitigate | ConnectionError → honest 503 "search unavailable" (the neo4j-503 mirror); never a crash or a fake empty list |
| T-10-21 | Information Disclosure | secrets/tokens leaking into the ES index or logs | mitigate | the structlog redaction processor masks password/secret/token/credential before render; indexed docs carry no secret fields |
| T-10-22 | Tampering | a slop/typosquat package via the install | mitigate | T-10-SC — the elasticsearch install is GATED behind a blocking checkpoint:human-verify (official Elastic client; client major == server 9.x) |
| T-10-SC | Tampering | npm/pip install (elasticsearch) | mitigate | blocking human checkpoint before `uv add`; verified on pypi.org/project/elasticsearch + the 9.x server-match |
</threat_model>
