<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser nav gating | UX-only role hiding; never the security boundary |
| dashboard page → API | a directly-typed forbidden URL must render no-access, never the data |
| QA artifact links | must be auth-gated URLs from run-relative basenames, never raw filesystem paths |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-23 | Elevation of Privilege | relying on nav-hiding as the control | mitigate | nav gating is UX; the API require_role (Plan 01-02) is the boundary; a forbidden URL → the API 403s → the page renders no-access, never the data |
| T-10-24 | Tampering | path traversal via a QA artifact link | mitigate | links are built as auth-gated URLs from the run-relative basename (the Phase-7 route + its containment guard); the page never constructs an fs path |
| T-10-25 | Information Disclosure | a fabricated metric/percentage shown without server data | mitigate | every KPI/chart/row renders strictly from the server payload; loading shows skeletons, never a fabricated number (the honesty rule) |
| T-10-26 | Information Disclosure | conflating the two coverage numbers | mitigate | the executive dashboard renders them as separate tiles each with its definition (Pitfall 5) |
</threat_model>
