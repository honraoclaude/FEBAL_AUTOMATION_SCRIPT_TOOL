# Phase 10 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Phase 10 UI surface (PLAT-04 + DASH-01..06) — MULTI-SURFACE, all in `app/(dashboard)/`:** this IS the dashboards phase. It ships SIX role-gated surfaces over the EXISTING Phase-4..9 data, plus the role-gated nav and a minimal admin role-assignment screen. No new domain capability — everything VISUALIZES, GATES, or SEARCHES what Phases 4-9 already produced. The surfaces:

1. **Executive dashboard** (`/dashboards/executive`) — DASH-01: KPI tiles up top (coverage %, pass rate, defect counts) + Recharts trend cards below (pass-rate over time, defect trend over time). Shows BOTH coverage numbers, each with its honest definition, never conflated.
2. **QA dashboard** (`/dashboards/qa`) — DASH-02: an execution-history table (the Phase-7 styled `table` precedent) + a failed-tests panel + screenshot/video links built as AUTH-GATED URLs from run-relative basenames (NEVER raw filesystem paths — the Phase-7 contract).
3. **Developer dashboard** (`/dashboards/developer`) — DASH-03: root-cause groupings (by the Phase-9 classification + fingerprint), an error-trend chart, and a module-failure-breakdown table/bar (failures grouped by flow/page).
4. **Coverage panel** (`/coverage`) — DASH-04: the coverage % + the honest definition string DISPLAYED inline + a per-flow drill-down table. The Phase-5 ground-truth coverage is shown SEPARATELY with its own definition (the two are distinct, each labeled — never merged into one number).
5. **Traceability viewer** (`/traceability`) — DASH-05: pick any artifact (flow / scenario / run / defect) → render the flow↔scenario↔script↔execution↔defect chain with drill links to the existing detail pages; honest "no link" segments where a chain link does not exist.
6. **Search UI** (`/search`) — DASH-06: a query box over executions / failures / logs (ES-backed) → a typed result list with per-hit type badge + highlighted match; honest "search unavailable" when the `search` profile is down (mirrors the graph-down 503), plus empty + loading states.

**Plus two cross-cutting contracts:**
- **Role-gated nav + views (PLAT-04):** the sidebar renders only the items the current role (from `/me`) may use; an admin-only **role-assignment screen** (`/admin/users`) lists users + sets each one's role. Honest "no access" states; the UI NEVER renders data the role can't see (server-enforced via `require_role`; the UI mirror is UX-only, never the security boundary).

**Design system is LOCKED by 01-UI-SPEC.md, extended by 04/05/06/07/09-UI-SPEC, and confirmed in `apps/web/components.json` + `app/globals.css` + the vendored `apps/web/components/ui/` on disk this session — it is REUSED verbatim here, not redefined.** This document specifies only what is NEW for Phase 10: the six page surfaces + the admin screen, the KPI tile / chart-card / table / chain / search-result treatments, the role-gating model, the two-coverage-metric discipline, the artifact-link contract, every empty / loading / error / honest / no-access state, and the sidebar nav changes — all in the established voice. Every spacing/typography/color value below REFERENCES the Phase-1 tokens.

**AUTO-mode note:** No design-system questions were asked — 01-UI-SPEC (the locked tokens), 07-UI-SPEC (the Recharts trend-card precedent + honest run states + sidebar-append + artifact-link contract), 09-UI-SPEC (the most recent approved: list+detail, honest states, word+icon discipline, zero-new-dep), 10-CONTEXT (D-01..D-04), and 10-RESEARCH (the coverage join, traceability chain shape, ES search contract + graceful-degrade, the role→permission map, the flow_id-positional honest-definition note) already answered every contract question. The codebase was scanned this session: `apps/web/components.json` (preset), `app/globals.css` (the `--status-*` tokens + zinc palette), `apps/web/package.json` (recharts 3.8.1 + @tanstack/react-query PRESENT; **@tanstack/react-table NOT present and NOT used anywhere** — see the dependency note), `components/app-sidebar.tsx` (the `NAV_ITEMS` flat-list contract + the `/me` query), and the vendored `apps/web/components/ui/` (16 blocks). Sources are cited per section; anything not sourced is a documented default.

Do NOT add `@tanstack/react-table` — that would break the zero-new-dep discipline for a capability the existing tables already deliver without it.

> **Coverage / pass-rate meters — token-styled native, NOT a Recharts gauge:** a single percentage (coverage %, pass rate as a KPI) reads as a **mono numeral + a token-styled native meter** (a `bg-secondary` track with a `--status-pass` fill), the Phase-9 confidence-meter / Phase-7 progress-bar precedent — a styled-native `role="progressbar"` over tokens, not a chart. Recharts is reserved for TRENDS (a series over time), not single scalars.

**Chain-segment present-vs-missing (the traceability viewer):**

|-------|-------|-----------|

### Accessibility

- **Tables:** real `<table>` semantics via shadcn `table` (`<th scope="col">`). Role, search-type, verdict, class, and coverage-state badges are NEVER color-only — each carries its WORD (role / type / verdict / class / "Covered"/"Not covered") alongside the colored dot/icon (WCAG 1.4.1); the dot/icon is `aria-hidden`.
- **Coverage/pass-rate meters:** real `role="progressbar"` with `aria-valuenow`/`aria-valuemin="0"`/`aria-valuemax="100"` and an accessible name combining the value and what it measures (e.g. `aria-label="Coverage 62 percent of discovered flows"`); the mono numeral is always present as text.
- **Charts (Recharts):** each chart `Card` has a caption title + an accessible summary (`aria-label` / visually-hidden text, e.g. "Pass rate over the last {n} runs"); the chart is supplementary — the KPI/table is the source of truth (WCAG 1.4.1 non-text).
- **Traceability chain:** rendered as an ordered list (`<ol>`) so the flow → scenario → script → execution → defect order is conveyed structurally; each node is a labeled region; a missing segment is real text ("No defect linked."), never an empty or dead node.
- **Search:** the query box is a labeled `<input type="search">`; the results region is a `role="region"` with an `aria-label` ("Search results"); the result count is `role="status"` `aria-live="polite"` so "{n} results" is announced after a search; each highlighted snippet keeps the matched term as real emphasized text.
- **Admin role control:** the per-row `dropdown-menu` is keyboard-operable (shadcn default); the confirm dialog is a `dialog` with focus trapped and "Keep {current role}" as the default focus; the disabled self-row carries the "You can't change your own role." text as its accessible explanation.
- **No-access region:** rendered as text with a reachable link to a permitted home; never icon-only or color-only.
- **Empty/error regions:** rendered as text with a reachable action link/button (Go to executions / Go to targets / Retry); never icon-only or color-only.
- **Motion:** any meter/chart entrance animation respects `prefers-reduced-motion` (drop to instant when set), the inherited motion rule.

### Interaction defaults (inherited from 01-UI-SPEC, applied here)

- Focus rings: 2px accent ring on all interactive elements (the search box, filter/type segments, drill-in links, artifact links, the Recharts-card links, the admin role `dropdown-menu`, Retry).
- Sentence case throughout; verbs-first on buttons ("Search", "Show chain", "Change role", "Retry").
- Errors render inline in the content region, never as toasts; the single success toast (sonner, bottom-right) is the admin role change.
- No optimistic updates — every metric, chain, search result, and role is server-authoritative; the UI repaints from the server response after the one mutation (role change).
