<objective>
Build the read-time cross-store traceability join (DASH-05): given ANY artifact id — a flow_id,
scenario_id, run_id, or defect_id — resolve and assemble the full
flow ↔ scenario ↔ script ↔ execution ↔ defect chain by joining Neo4j flows (via kg/reader) with
the Postgres lifecycle tables (already FK-linked by run_id + flow_id from Phase 9). NO new graph
writes — the single-writer Neo4j discipline holds; relational lifecycle data is never coupled into
the KG. After this slice, the viewer (Plan 05) can render the chain for any picked artifact, with
honest gaps where a link does not exist.

Purpose: DASH-05 — the traceability engine, keyless + deterministic + fixture-testable.
Output: traceability service + schema + role-gated router + registration + the entry-by-any-id
join tests.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → traceability API | role-scoped read access; lifecycle data must not leak to a disallowed role |
| API → Neo4j + Postgres | a read-time cross-store join; must add no graph writes (single-writer discipline) |
| entry id → store queries | a user-supplied artifact id is a query VALUE, never interpolated into Cypher/SQL |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-12 | Elevation of Privilege | GET /api/traceability | mitigate | router-level `require_role("admin","qa_lead","developer")`; 403 matrix asserted |
| T-10-13 | Tampering | violating the single-write-path discipline via traceability | mitigate | the service joins on READ only — adds NO write-Cypher; the single-write-path grep gate stays green |
| T-10-14 | Tampering | injection via the entry id | mitigate | entry ids are parameterized query values (SQLAlchemy params; kg/reader's parameterized Cypher) — never string-built |
| T-10-15 | Information Disclosure | a fabricated chain link implying a relationship that does not exist | mitigate | missing segments render as honest empty/null gaps; only server-confirmed links appear |
| T-10-16 | Denial of Service | graph-down 500ing the chain | mitigate | the flow segment degrades to null + a note; the relational chain still assembles |
</threat_model>
