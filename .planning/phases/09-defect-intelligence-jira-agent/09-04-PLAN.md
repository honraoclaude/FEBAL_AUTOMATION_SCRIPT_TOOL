<objective>
Wire the defect pipeline end-to-end behind the keyless `JiraGateway` (Plan 03) and the pure classifier (Plan 01): after a product-failure classification, persist a draft `Defect` row with the fingerprint + the test↔flow↔execution traceability links (JIRA-04), then file-or-update to Jira via fingerprint-label JQL dedup (JIRA-03) under the per-run create cap, gated by the OFF-by-default autonomy flag + the calibrated confidence threshold (JIRA-02). Expose the auth-gated `/api/defects` review router (list/detail/apply/reject/calibration) that Plan 05's UI consumes. Every Jira interaction routes through the gateway so the whole flow is FakeJira-testable — keyless; live filing/dedup is Manual-Only.

Purpose: JIRA-02 (draft/review queue + the autonomous-filing gate, OFF by default, only above the calibrated threshold), JIRA-03 (fingerprint + JQL dedup, update-not-duplicate, per-run cap), JIRA-04 (every defect linked test↔flow↔execution via the Defect row FKs + the Jira issue link). The decision is already pure (Plan 01); the threshold is already calibrated (Plan 02); the gateway is already keyless-contracted (Plan 03) — this plan composes them into the draft-queue lifecycle + the auth-gated review API.
Output: `defects/pipeline.py` (the orchestrator), `defects/autonomy.py` (the structural gate), `schemas/defect.py`, `routers/defects.py` (registered in main.py), and the keyless dedup/autonomy/pipeline/router tests against FakeJira.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /api/defects router | untrusted HTTP (incl. state-changing apply/reject/autofile) crosses into defect filing |
| platform → Jira Cloud (external) | issue create/comment/attach/link/JQL cross to a third-party with a secret token |
| server fingerprint → JQL label | the dedup query is built from a server-side hash; must carry no user-controlled text |
| stored artifact paths → Jira attachment + detail links | untrusted error text + artifact paths flow into the issue body + attachment streaming |
| confidence + autonomy flag → autonomous filing | the structural gate decides whether a real ticket is filed without a human |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
