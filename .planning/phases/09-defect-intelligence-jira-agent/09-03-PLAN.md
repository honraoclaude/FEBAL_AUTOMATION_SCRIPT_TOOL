| Boundary | Description |
|----------|-------------|
| platform -> Jira Cloud (external) | issue create/attach/JQL/link cross to a third-party over the network with a secret token |
| evidence/artifact paths -> ADF + attachment | untrusted error text + artifact paths flow into the issue body + attachments |
| async FastAPI -> sync atlassian client | a sync call inside the event loop blocks all requests |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-08 | Information Disclosure | jira_api_token in logs/errors | mitigate | the token is boot-safe optional + NEVER logged (the ci_token / anthropic_api_key SENSITIVE-regex precedent); no structlog event in jira/ carries url/username/password/token |
| T-09-09 | Denial of Service | sync client blocking the event loop | mitigate | every atlassian call is wrapped in anyio.to_thread.run_sync (Pitfall 3) — the gateway is async, the offload is cancellation-aware |
| T-09-10 | Tampering | a string description rejected by Cloud v3 | mitigate | build_adf returns a doc DICT (Pitfall 2/6); api_version="3" + cloud=True; the unit test asserts the dict shape |
| T-09-11 | Supply chain | the gated dependency | mitigate | atlassian-python-api install is a blocking-human checkpoint (Task 1) verifying pypi/source/version; [ASSUMED] until verified; the CLAUDE.md-locked choice, not the `jira` pycontribs pkg |
