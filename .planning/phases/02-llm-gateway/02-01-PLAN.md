<objective>
Slice 1 of the LLM Gateway: a provider-agnostic `complete()` call that routes through `init_chat_model`, reads usage uniformly across Anthropic and OpenAI, computes USD cost from an effective-dated pricing table, and records every call to a durable `llm_usage` Postgres ledger plus a redacted structlog usage event. This proves PLAT-05 minimally end-to-end: a metered LLM call in, a cost row out.

Purpose: Establish the single money-control chokepoint's call path and cost-accounting spine before any budget enforcement (Plan 02) or caching (Plan 03) layers on top. Every future agent imports this gateway as its ONLY LLM path.
Output: New deps (gated), settings env contract, pricing module, ledger model + migration, the gateway service with `complete()`, unit + functional tests, and the Wave-0 mocked test scaffold + `live_llm` marker.
