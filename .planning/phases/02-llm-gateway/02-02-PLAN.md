<objective>
Slice 2 of the LLM Gateway: enforce per-call / per-run / per-day token+USD budgets as a PRE-CHECK before spend, reconcile to actual usage after the call, and provide a hard kill-switch that halts all LLM traffic — tripped both manually (authenticated admin endpoint) and automatically on daily-budget exhaustion. Closes PLAT-06's budget + kill-switch half.

Purpose: No agent (Phase 4+) can overspend even by a single call, and a panic button can stop everything instantly. Builds directly on Plan 01's `complete()` call/accounting path.
Output: A lifespan-managed shared Redis client, budget env settings, the pre-check/reconcile/auto-trip logic inside `complete()`, the admin kill-switch router, and unit + functional tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller → complete() | run_id, model, run-budget-override params are untrusted; overrides must not loosen caps |
| client → admin kill-switch endpoint | external HTTP; must require auth |
| gateway/admin → Redis | budget counters + kill flag shared across api/workers |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07 | Tampering | budget bypass via loose per-run override | mitigate | Clamp run overrides to min(override, global cap); never loosen (RESEARCH Q4); unit test asserts clamp |
| T-02-08 | Elevation of Privilege / DoS | unauthenticated kill-switch toggle | mitigate | Router-level Depends(get_current_user); 401 asserted in functional test; Admin-RBAC gap documented (Phase 10) |
| T-02-09 | Tampering | counter race → minor overspend under concurrency | accept | Read-then-check accepted for single-user MVP; documented in code (RESEARCH Pattern 3 race note); upgradeable to reserve-via-INCR if needed |
| T-02-10 | DoS | daily budget exhausted but counter never resets | mitigate | UTC date-bucketed day key self-resets; 2-day TTL GCs old buckets (Pitfall 4) |
| T-02-11 | Repudiation | budget breach silently ignored | mitigate | BudgetExceeded raised (typed exception, D-02); callers fail fast, not a silent result object |
| T-02-12 | DoS | kill-switch fails to halt traffic | mitigate | Kill check is the FIRST hot-path op (GET before pre-check/provider); functional test proves halt |
</threat_model>
