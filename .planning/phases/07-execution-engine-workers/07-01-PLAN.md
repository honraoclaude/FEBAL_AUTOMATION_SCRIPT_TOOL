<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| API → RabbitMQ | the producer publishes job messages the worker trusts |
| RabbitMQ → Worker | a job payload crosses into subprocess argv construction |
| Worker → OS subprocess | spec_path becomes a pytest argv element |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | worker/job.py subprocess argv (spec_path) | mitigate | argv LIST + no shell=True; spec_path is run_id-derived (run_dir(run_id)/...), NEVER a raw value from the message body (carry-forward T-03-15) |
| T-07-02 | DoS | hostile/poison job payload causing infinite redelivery | mitigate | message.process(requeue=True) bounded; malformed JSON → caught, message acked/dead-lettered not infinitely requeued; prefetch_count=2 caps in-flight work |
| T-07-03 | DoS | resource exhaustion (too many browsers under the 3GB cap) | mitigate | prefetch_count=2 hard bound (settings); worker mem_limit 768m; neo4j off during runs (documented sequencing) |
| T-07-04 | Elevation/Governance | LLM call sneaking into the worker (SC3 breach) | mitigate | tests/unit/test_no_llm_in_worker.py import grep gate over app/services/worker/ + worker_main.py |
| T-07-SC | Tampering | aio-pika install (supply chain) | mitigate | blocking human-verify checkpoint (Task 1) verifying pypi.org/project/aio-pika before install; pinned 9.6.* |
</threat_model>
