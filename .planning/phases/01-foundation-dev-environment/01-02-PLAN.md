## Phase Goal

**As a** platform operator, **I want to** stand up the platform skeleton locally with one command, log in, and securely register a target application, **so that** the platform has a target to explore in later phases.

(This plan adds the API tier to the one-command stack: config, logging, DB plumbing, migrations, /health — the chassis the auth and registry slices bolt onto.)

<objective>
Create the FastAPI application skeleton as a uv project: typed settings, structlog with credential redaction, async SQLAlchemy engine/session, async Alembic, a /health endpoint pinging Postgres and Redis, the api Dockerfile (self-migrating entrypoint per D-08), the compose `api` service, and the pytest Wave 0 scaffolding.

Purpose: every API feature (auth, targets) builds on these exact modules; the redaction processor is a PLAT-07 control; conftest is the Wave 0 gap from 01-VALIDATION.md.
Output: api container healthy in `docker compose up`; `uv run pytest` collects with asyncio_mode=auto.
</objective>
