## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| repo → git remote | secrets must never cross into committed history |
| host → containers | env-var injection is the only secret path into containers |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | .env secrets committed to git | mitigate | `.env` in `.gitignore` from first commit; `.env.example` ships placeholders only; acceptance criterion asserts `git check-ignore .env` |
| T-01-02 | Information Disclosure | Encryption/JWT keys as literals in compose YAML | mitigate | compose references `${VAR}` interpolation only; acceptance criterion greps compose for literal secret values |
| T-01-03 | Denial of Service | Unbounded container memory starving Windows host | mitigate | `mem_limit` on every service incl. dormant; `docker inspect` non-zero check; `.wslconfig` cap (verified at phase gate) |
</threat_model>
