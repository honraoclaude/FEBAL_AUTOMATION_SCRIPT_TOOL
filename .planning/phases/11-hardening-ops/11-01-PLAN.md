<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Prometheus scraper → api /metrics | A scraper (and anything on the network) can GET /metrics unauthenticated |
| refresh task → DB/Neo4j | The snapshot reader touches Postgres + Neo4j every 30s |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Information Disclosure | /metrics endpoint | accept | Emit ONLY aggregate numeric gauges + HTTP histograms — no prompts/credentials/PII (llm_usage has no prompt/response columns, PLAT-07). Unauthenticated-but-safe, the root /health precedent; single-operator local posture (A4). A scrape-token gate is deferred (documented). |
| T-11-02 | Denial of Service | /metrics scrape path | mitigate | collect() reads cached floats O(1) — NO DB I/O on the scrape path; the 30s background refresh is the only DB load. A flood of scrapes cannot amplify DB load. |
| T-11-03 | Denial of Service | metric refresh loop | mitigate | Each source independently try/excepted → a down source sets None, never raises; /metrics always 200. A down Neo4j never flaps up{job=api} (RESEARCH Pitfall 2). |
| T-11-04 | Tampering | dashboard JSON / prometheus.yml | accept | Config-as-code in git, reviewed; no runtime mutation surface. promtool + JSON-valid gate catches malformed config before deploy. |
| T-11-SC | Tampering | pip install (2 prometheus deps) | mitigate | slopcheck [OK] in RESEARCH audit + blocking-human checkpoint (Task 1) before install. |
</threat_model>
