<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| manifests in git → cluster | Committed YAML defines what runs and with what credentials |
| Secret → pods | Sensitive env is delivered to api/worker/neo4j/postgres |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-10 | Information Disclosure | secrets in manifests | mitigate | Sensitive env lives in a K8s Secret (NOT a ConfigMap, NOT inline in Deployments) with placeholder/example values committed (secret.example.yaml); real values gitignored (Security V14). |
| T-11-11 | Denial of Service | resource exhaustion | mitigate | Every workload sets resources.requests + limits from the compose mem_limits; the README documents the 3GB fit (ES off + monitoring separate + scale-web-to-0 during explore) so the deploy cannot OOM the node (RESEARCH Pitfall 4). |
| T-11-12 | Tampering | malformed manifests reach a cluster | mitigate | kustomize build | kubeconform -strict gate (validate.sh + CI) catches schema errors offline before any apply — no apply-to-discover-errors. |
| T-11-13 | Information Disclosure | /metrics exposure in-cluster | accept | /metrics is a ClusterIP-scoped aggregate-numeric surface (Plan 01 T-11-01); no Ingress/TLS in scope (single-operator local validation, deferred). |
| T-11-14 | Spoofing | wrong image pulled | mitigate | api/web reference the GHCR images from Plan 02 (ghcr.io/honraoclaude/...); worker reuses the api image — no untrusted third-party image in the core set (exporters are the official upstream images per the CLAUDE.md table). |
</threat_model>
