## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| approved scenario → codegen | approved gherkin + then_refs feed template rendering |
| LLM slot fills → rendered .py | narrow LLM slots land in generated source — scanned before write |
| Element Repository → page objects | repo locators are the only sanctioned literal source |
| generated .py → filesystem (workspaces) | written only after ast.parse + selector gate pass |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-12 | Tampering | freehand/malicious selector in generated code | mitigate | static AST selector gate rejects inline literals in spec/step modules; page objects are the only literal home and their literals are repo-traceable (asserted); no partial write on violation |
| T-06-13 | Tampering | non-importable / injected code in a rendered .py | mitigate | ast.parse every render before write; a parse failure raises GenerationError and writes nothing (Phase-3 invariant) |
| T-06-14 | Elevation of Privilege | codegen consuming non-approved scenarios | mitigate | generate_project reads scenario_service.list_approved only (D-01) |
| T-06-15 | Spoofing | unmetered LLM spend in codegen | mitigate | codegen structure/selectors are deterministic (Jinja2 + repo); any LLM slot-fill routes the gateway (operation_type+run_id) — no init_chat_model |
| T-06-16 | Info Disclosure | target creds in generated artifacts | mitigate | SauceDemo public demo creds only; repo locators carry no secrets; no decrypted/ciphertext creds in templates (PLAT-07) |
| T-06-17 | Tampering | writing to Neo4j during locator reads | mitigate | locators.py uses reader.element_repository (execute_read); single-write-path grep gate stays green |
| T-06-SC | Tampering | npm/pip/cargo installs | mitigate | ZERO new packages (pytest-bdd/jinja2/playwright already installed); no new pin; any new package requires a blocking-human checkpoint |
