<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → POST /explore | authenticated request with target_id (untrusted input) |
| explorer → SauceDemo | browser drives the registered target with decrypted creds |
| explorer → Neo4j | page-derived strings written via Cypher |
| BackgroundTask → Postgres/Neo4j | runs after response; owns its own session |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Tampering | Cypher MERGE on page-derived strings | mitigate | parameterized Cypher ONLY (`$a_key`,`$a_url`); never f-string page text into the query (Pattern 2) |
| T-03-06 | Info Disclosure | decrypted target credentials | mitigate | creds obtained only via `get_decrypted_credentials`; never written to Page nodes, never logged (PLAT-07; structlog redaction) |
| T-03-07 | Spoofing | unauthenticated /explore or /executions | mitigate | router-level `Depends(get_current_user)` on both; auth-gate test asserts 401 |
| T-03-08 | Tampering | SSRF via explore target URL | accept (scoped) | tracer crawls the registered SauceDemo target only; full origin_allowlist enforcement is Phase 4 EXPL-08 |
| T-03-09 | Repudiation | run status integrity | mitigate | status transitions only via run_service guarded by VALID set; failure captured as `error` not a silent crash |
</threat_model>
