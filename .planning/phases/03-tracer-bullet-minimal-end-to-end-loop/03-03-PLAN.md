<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → POST /generate-* | authenticated request with run_id |
| service → LLM provider (via gateway) | prompt + generated code crossing to/from the provider |
| service → filesystem | writes .feature + .py under workspaces/ |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-10 | Info Disclosure | provider key / target creds in generated code | mitigate | generation routes through the gateway (keys never enter prompts/artifacts); SauceDemo public demo creds used as literal template values, never target ciphertext (PLAT-07) |
| T-03-11 | Tampering | LLM prompt injection from page content | mitigate (scoped) | tracer feeds only the deterministic OBSERVED selectors to the LLM, not raw page DOM; full untrusted-input sanitization is Phase 4 EXPL-08 |
| T-03-12 | Tampering | malformed/invalid generated artifacts | mitigate | gherkin-official validates the .feature before write; Jinja2 owns spec structure so the LLM cannot emit arbitrary control flow |
| T-03-13 | Spoofing | unauthenticated /generate-* | mitigate | router-level `Depends(get_current_user)`; auth-gate test asserts 401 |
| T-03-14 | Denial of Service | unbounded LLM spend | mitigate | gateway enforces per-call/run/day budgets + kill-switch (PLAT-06, inherited) |
</threat_model>
