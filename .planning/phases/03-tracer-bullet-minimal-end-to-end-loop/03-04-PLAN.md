| client → POST /execute | authenticated request with run_id |
| service → subprocess (uv run pytest) | a generated spec is executed as a child process |
| service → Postgres | result row written from a fresh session |
| client → stub endpoints | authenticated requests to not-yet-built surface |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-15 | Elevation | subprocess command injection via spec_path | mitigate | `create_subprocess_exec` argv list, no `shell=True`; spec_path is run_id-derived (`workspaces/<run_id>/test_login.py`), never raw user input (T-01-26) |
| T-03-16 | Tampering | running generated spec in the API process | mitigate | subprocess only — never `pytest.main()` in-process (Pitfall 3); the spec runs isolated |
