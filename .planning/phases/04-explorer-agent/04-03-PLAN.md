<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM decide output → act | the LLM-chosen index crosses into execution — the code gate sits here |
| page content → decide prompt | untrusted page text enters the LLM context |
| target.origin_allowlist → navigate | code decides whether a goto is in scope |
| page-derived locator/validation strings → Neo4j | written as node properties |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-11 | Tampering/Elevation | destructive action on a non-sandbox target | mitigate | Code-enforced deny-list (is_destructive) evaluated BEFORE the act; sandbox flag required to lift (D-03); never LLM judgment |
| T-04-12 | Elevation (scope escape / SSRF-ish) | off-origin navigation | mitigate | Code-enforced is_off_origin check before goto, reads Target.origin_allowlist (D-04) |
| T-04-13 | Tampering/Elevation | prompt injection via page content | mitigate | Untrusted-observation delimiting (data-only system prompt) PLUS the deterministic risk gate AFTER the decision — defense in depth (Pitfall 5); injected LLM cannot trigger destructive/off-origin |
| T-04-14 | Tampering | Cypher injection via locator/validation/page strings | mitigate | Parameterized Cypher MERGE only; chain/history/validation serialized as JSON params, never f-string into the query (T-03-05) |
| T-04-15 | Integrity | Element/Workflow no-op write | mitigate | Managed execute_write + read-back guard fails the run on a 0-count write (SC1 lesson) |
</threat_model>
