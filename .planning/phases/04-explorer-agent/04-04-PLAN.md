<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser EventSource → /api/explore/{run_id}/events | unauthenticated network reaching a streaming endpoint |
| screenshot path → browser <img src> | a run-scoped filesystem path served to the client |
| Redis pub/sub channel | per-run progress events fan out |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-16 | Spoofing/Access | SSE events endpoint | mitigate | Router-level Depends(get_current_user); EventSource can't set headers so cookie auth (httpOnly, same-origin proxy) is the only auth — the gate is server-side, verified by a 401 test |
| T-04-17 | Info Disclosure / path traversal | screenshot served to <img> | mitigate | Screenshot path is run_id-derived under workspaces/<run_id>/ (never client input); if served via an /api/explore/{run_id}/screenshot route it is auth-gated and the filename is validated against the run's workspace (no `..`) |
| T-04-18 | DoS | a client holding the SSE stream open | mitigate | request.is_disconnected() break + finally unsubscribe releases the pubsub; single concurrent run cap (memory Pitfall 2) |
| T-04-19 | Info Disclosure | cost_usd / event payload | accept | Events carry no creds/PII; cost is an aggregate USD figure from the gateway counter; feed lines are page labels (already shown in the graph) |
| T-04-20 | Tampering | injected page label rendered in the feed | mitigate | feed_line rendered as React text (auto-escaped), never dangerouslySetInnerHTML; refused-reason is plain text |
</threat_model>
